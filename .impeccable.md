# Design Context

## Users
Oracle DBAs, performance engineers, and senior database consultants looking up Tanel Poder's TPT troubleshooting scripts during diagnostic work. They're veterans — they run scripts in SQL*Plus from `~/scripts`, know what `v$session` is, and have been doing this for 10+ years. Context of use: a second monitor during an incident, a laptop on a plane, or the office at 2am. Not beginners. Not browsing for fun.

## Job to be done
1. "I need to find the right script fast" — fuzzy recall ("there was one for latch profiling...")
2. "Show me what's available in this category" — category browsing when learning the toolkit
3. "Let me read the source" — verify the script does what they expect before running it

## Brand personality
Three words: **veteran, dense, referential**. This is reference material from a recognized expert. The UI should feel like it honors the content — a well-organized toolbox, not a marketing landing page.

## Aesthetic direction
Currently supports 3 themes with dark/light mode. User feedback: the current themes feel templated/generic (Copper reads as Linear-clone, Daylight as Stripe-docs-clone, Brutalist as generic-punk-zine). Information density is off. Typography is bland.

Scope: restructure freely. Each theme should have real conceptual identity.

Refined direction for each theme:
- **Bureau** (was Brutalist) — an engineering bureau's indexed filing cabinet. Warm ink on cream, careful, considered, marginalia-rich. Not punk zine.
- **Console** (was Copper) — professional tool in the Bloomberg-terminal / finance-app lineage. All monospace, dense, status-barred.
- **Atelier** (was Daylight) — a designer's studio: generous, organic, slightly asymmetric. Not corporate SaaS clean.

## Anti-references
- Generic SaaS dev tools (Linear, Notion, Stripe docs)
- "Modern" dashboards with pill badges everywhere
- Reflex fonts: Inter, Outfit, DM Sans, IBM Plex, JetBrains Mono, Space Grotesk, Fraunces
- Gradient text, side-stripe border accents, purple-on-dark neons
- Centered hero blocks with big watermark numbers
- "Cyan-on-dark" AI palette

## Design principles
1. **Content earns the pixels** — every card should show information, not decoration. Dense beats airy.
2. **Intentional typography** — pick fonts with character. Real hierarchy (1.25+ ratio). Exploit OpenType features (tabular nums, ligatures).
3. **Three themes, three identities** — each should feel like a different person designed it, not three presets of one template.
4. **No templated patterns** — no pill-everywhere, no side-stripe active states, no centered hero. Break the 2024 dev-tool mold.
5. **Respect the audience** — these users don't need handholding. Keyboard-first, dense, assumes competence.
